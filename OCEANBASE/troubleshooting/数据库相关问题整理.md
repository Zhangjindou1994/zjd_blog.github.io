# 数据库相关问题整理

## 功能问题

### 报错OBE-04038 无主

```
问题现象:
城乡居保待遇发放SP执行报错，报错信息:OBE-600:internal error code,argument:-4038,the observer or zone is not the master
问题原因:
事务执行过程中，对复制表进行了DML操作，导致复制表行为回退到普通表，并且事务会反复寻找复制表的leader节点导致事务报错
解决方法:
修改事务逻辑使得复制表不在事务当中进行增删改或者将复制表修改为普通表
```

### 报错OBE-04013内存超限

```
问题现象:
月度批量处理某核心sql执行报错，ErrorCode = 600, SQLState = HY001, Details = OBE-00600: internal error code, arguments: -4013, No memory or reach tenant memory limit
问题原因:
sql中有一段复杂函数排序逻辑，
XMLAGG(XMLPARSE(CONTENT AAZ172 || ',') ORDER BY AAZ172).GETCLOBVAL()，
在当前OB 4.3.5.1 hotfix5中，该语法暂不支持数据落盘，因此sql_work_area会持续膨胀至报错
解决办法:
修改语法去除排序部分/sql增加hint no_use_hash_aggregation/升级版本至4.3.5.1 hotfix7以上
```

### 报错OBE-01000

```
问题现象：
业务sql执行报错:
OBE-01000: maximum open cursors exceeded; nested exception is java.sql.SQLTransientConnectionException: (conn=566423) OBE-01000: maximum open cursors exceeded",
问题原因:
jdbc使用cachePrepStmts=true参数之后，如果有不使用绑定变量的sql执行后，会导致会话的ps句柄数大量累积，累积到系统参数open_cursors设置的阈值之后会导致报错
解决办法:
临时修改参数解决/业务修改不使用绑定变量的sql
```



### sql重复执行结果不一致

```
问题现象：
业务有关sql重复执行结果不一致
问题原因:
ob当前版本优化器存在bug
解决办法:
修改优化器参数:
alter system set _adaptive_join_enabled = false;
```



## 性能问题

### 优化器执行计划生成错误或不佳

```
问题现象:
当前版本4.3.5.1中，部分复杂sql会生成错误或效率低下的执行计划或者发生执行计划跳变导致业务执行效率变慢
解决办法:
针对性的添加hint以及对特定sql通过outline绑定执行计划
```



### 路由错误

```
问题现象:
应用sql通过obproxy代理没有分发到正确的observer节点
解决办法:
1.当前版本obproxy代理不支持分区条件为函数值，例如分区键为aac001，使用where aac001=to_number(111)无法正确路由
2.匿名块和存储过程路由会随机分发，需要按需指定特定的observer执行
```



### 单节点并行

```
问题现象:
月处理sql中，部分并行sql在特定observer节点执行，并行数过高导致节点cpu 100%
解决办法:
控制sql并行度在可控范围内
```



### 部端原子服务更新接口耗时长

```
问题现象:
本地进行信创改造之后，对表的分区进行了改造，原oracle中的全局唯一主键，修改为本地唯一主键之后，相关表上的原子服务更新接口执行效率降低
解决办法:
1.本地表增加原主键的全局唯一分区索引，但该方式会增加额外索引且效率达不到主键更新的速度
2.部端接口增加主键自定义参数
```



## 组件问题

### jdbc驱动bug

```
问题现象:
应用执行部分insert 类型sql时报错，导致业务终端
问题原因:
当前使用的jdbc 2.4.15版本驱动存在bug
解决方法:
厂商更新驱动版本修复该bug
```



### OMS迁移工具bug

```
问题现象:
使用ob的官方迁移工具OMS平台迁移数据表结构时，发现迁移效率较慢，并且存在部分表结构丢失索引的现象
问题原因:
当前ob版本在并行ddl时，可能会出现4012超时现象，导致oms迁移报错并跳过该索引创建
解决办法:
ob侧修改ddl并行参数，关闭并行索引创建
OMS侧升级版本，由4.2.4升级到4.2.5
```



## 配置问题

### jdbc连接断链

```
问题现象:
应用在特定情况下稳定报错connection is closed
问题原因:
jdbc url中同时配置的loadbalance参数和数据库负载均衡地址
解决办法:
去除jdbc中的loadbalance参数，仅使用数据库负载均衡地址
```
