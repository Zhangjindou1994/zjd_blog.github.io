# 游标超限问题排查

## 问题现象

```
 OBE-01000: maximum open cursors exceeded; nested exception is java.sql.SQLTransientConnectionException: (conn=566423) OBE-01000: maximum open cursors exceeded"
```

## 有关参数释义

**open_cursors**

```
OceanBase 数据库中 open_cursors 参数，在 V3.2.4 (oceanbase-3.2.4.0-100000072022102819) 版本之前，控制的是一个 session 内打开的 ps cursor 数量，但是从 V3.2.4 (oceanbase-3.2.4.0-100000072022102819) 版本开始（不包含 V4.0.0 (oceanbase-4.0.0.0-100000262022110215) 版本），这个参数控制的是一个 session 内打开了 ps 句柄的数量加上 ps cursor 数量。
```

## 排查过程

首先查看打开的游标数，发现当前实际打开的游标数并不多

gv$open_cursor可以查看当前所有打开的游标数

```
SELECT
 sid,
 sql_text,
 cursor_type,
 last_sql_active_time
FROM
 GV$OPEN_CURSOR
WHERE
 con_id = 'xxx'
 and cursor_type='OPEN'
```

**ps句柄查询**

**查询ps句柄明细SQL**，可以查看指定租户，指定server上的句柄明细

```
SELECT
 ps_sql,
 param_count,
 mem_hold,
 stmt_item_ref_count
FROM
 oceanbase.GV$OB_PS_ITEM_INFO
WHERE
 tenant_id = 'xxx'
 AND svr_ip = '192.168.1.1';
```

 **关键字段说明 ：**
ps_sql：Prepare Statement 的 SQL 文本（截取前 60 字符）。
param_count：SQL 中参数的数量。
mem_hold：该 Prepare Statement 占用的内存大小（单位：字节）。
stmt_item_ref_count：该语句的引用计数，用于判断是否被频繁使用



**查询租户节点 ps cache整体状态**

```
SELECT
 svr_ip,
 svr_port,
 stmt_count AS ps_stmt_count,
 mem_hold AS ps_cache_mem_usage
FROM
 oceanbase.GV$OB_PS_STAT
WHERE
 tenant_id = 'xxx';
```

**关键字段说明 ：**

svr_ip / svr_port：OBServer 节点的 IP 和端口。

stmt_count：PS Cache 中缓存的 Prepare Statement 数量。

mem_hold：PS Cache 占用的内存大小（单位：字节）

**查询会话的ps 句柄信息**

```
select tenant_id,svr_ip,session_id,count(*) bs from __all_virtual_session_ps_info/gv$ob_session_ps_info
group by  tenant_id,svr_ip ,session_id order by 4  desc
```
